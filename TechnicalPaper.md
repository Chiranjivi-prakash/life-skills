## NoSQL Database

### What is a NoSQL Database?
- NoSQL stands for Not Only SQL
- It is used for storing large and flexible data
- It does not use fixed tables

### Types of NoSQL Databases
- Key-Value databases :- Store data as key and value pairs. Very fast and simple.
- Document databases :- Store data as documents like JSON. Flexible and easy to use.
- Column-based databases :- Store data in columns instead of rows. Good for large data and analytics.
- Graph databases :- Store data as nodes and relationships. Best for connected data.

### Brief History of NoSQL Databases
- NoSQL became popular with big data growth
- Used when traditional databases failed to scale

### NoSQL Database Features
- Schema-less design
- High scalability
- Fast read and write operations

### Relational Database vs NoSQL Database Example
- SQL uses tables and rows
- NoSQL uses flexible data formats

### Differences Between RDBMS and NoSQL Databases
- RDBMS has fixed schema
- NoSQL has dynamic schema
- RDBMS scales vertically
- NoSQL scales horizontally

### NoSQL Use Cases
- Social media platforms
- Real-time analytics
- Content management systems

## When Should NoSQL Be Used?
- Large amount of data
- High traffic applications
- Changing data structure

### NoSQL Database Misconceptions
- NoSQL does not replace SQL
- It is used for specific use cases

### NoSQL Query Tutorial
- Queries depend on the database type
- Usually simpler than SQL queries

### Summary
- NoSQL databases are flexible and scalable
- Widely used in modern applications
